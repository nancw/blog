    # 1、rtmpdump 在 cygwin64 编译拉未发布流阻塞

    # 问题描述

    • 在cygwin64编译设置接收超时，但没有达到效果，recv阻塞，同份代码在linux编译运行是正常的，超时后不阻塞。
    • 此处问题可引申，rtmpdump在cygwin64编译后从流媒体拉未发布的rtmp流会阻塞；ffmpeg在cygwin64编译后使用rtmpdump库拉未发布的流也会导致ffmpeg阻塞。

    # 问题思考

    • 可能是socket库的版本不同导致的问题，找到socket对应的库更换版本。

    # 问题说明

    • cygwin64的版本信息如下图：

    • rtmpdump编译，免编译openssl的修改配置。

    • rtmpdump的拉流命令：

      • ./rtmpdump -r rtmp://172.19.82.43/live -a live -y livestream -v -o 1.mp4
        1

    • ffmpeg的拉流命令

      • ffmpeg -loglevel debug -i "rtmp://172.19.82.228:1960/live/livestream live=1" -override_ffserver -flags:v +global_header -flags:a +global_header -acodec copy -vcodec copy http://172.19.82.43:7003/feed1.ffm
