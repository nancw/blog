    # 1、mp4文件流化

    # MP4流化的本质

    • 流化的本质是按顺序提取音视频数据帧的过程，即计算每个数据帧在文件中的偏移量的问题。
    • 因为MP4文件的音视频帧在文件中的物理位置不是按顺序存放的，而是一种相对凌乱状态存在。
    • 若在媒体播放过程中计算每帧偏移量难以达到高效率。故提出预处理的方案，即得到文件元数据时，将预先把每帧数据的大小和相对媒体文件的偏移地址计算出来。

    # 计算电影长度

    • 方法一，mvhd--timescale、duration，即电影时长=duration/timescale。例如：
      • audio track的time scale = 8000, duration = 560128，所以总长度是70.016；
      • video track的time scale = 600, duration = 42000，所以总长度是70；
    • 方法二，计算共有多少个帧，即sample（从sample size atom得到）。则电影时长=每个帧的duration之和（从time-to-sample atoms得到）。例如：
      • audio track共有547个sample，每个sample的长度是1024，则总duration是560128，电影长度是70.016；
      • video track共有1050个sample，每个sample的长度是40，则总duration是42000，电影长度是70；

    # 计算电影图像宽度和高度

    • 从tkhd--width、height，即电影图像的宽度和高度。

    # 计算电影声音采样频率

    • 从tkhd--audio track--mdia--minf--stbl--stsd--()--samplesize，即电影声音的采样频率。

    # 计算视频帧率

    • 帧率=帧的数目/电影总时长。

    # 计算电影的比特率

    • 比特率=电影文件长度/时长。
    • 例，源文件17 MiB，时长4分43秒，先将MiB转换成b，17(MiB)10241024*8 = 14260633(b)，随后将时间转换秒4分43秒 = 28秒，最后(142606336 / 283) * 1000 = 503.909..四舍五入后约 504(Kbps)，需要注意两点一个就是MiB和MB的区别，前者是以1024进位，而后者是以1000进位。
    • 为什么没有将MiB换算成Kib直接除以283秒计算比特率，原因就在于网络传输一般都是以1000进位的，如果不彻底将文件长度从MiB换算成b再计算的话，就会因为Kib中的K是以1024进位，而Kbps中的K是以1000进位的原因而影响到最后计算出来的比特率的准确性。
