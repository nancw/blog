    • 正数

      • 范围

    # 浮点数运算

      • M:尾数 R:基数 e:指数(阶码)
      • 浮点数的大小由阶码部分决定， 而其精度由尾数部分决定
      • 阶码为带符号的纯整数、尾数为带符号的纯小数

    • 运算步骤

      • 对阶、尾数计算、结果规格化

        对阶，两个数阶码相同，小阶向大阶看齐，较小阶码增加几位，尾数就右移几位。 尾数计算，相加，若是减运算，则加负数。 结果规格化，即尾数表示格式化，带符号尾数为1.0xxx或0.1xxxx

    • 浮点数

      • 浮点表示法

        • 阶符|阶码|数符|尾数

    • 异或，同0非1；逻辑左移，左移低位补0；算术左移，乘2运算

    • 溢出，正数相加为正溢，负数相加为负溢

      • 溢出判断方法
        • 表示当操作数符号位Sa和Sb都为0,且结果的符号位为1时，产生溢出。符号位Sa和 Sb都为0表示两个操作数均为正数，所以这种情况被称为"正溢" ；
        • 表示当操作数符号位Sa和Sb都为1,且结果的符号位为0时，产生溢出。符号位Sa和 Sb都为1表示两个操作数均为负数，所以这种情况被称为"负溢" ；
        • 在双符号位中，高位就是符号位的进位CF;而低位就是结果的符号位SF.我们可以看到，当CF与SF不同时，表示溢出；而相同时，表示操作正常。能达到此效果的运算为"异或"运算 。

    # 校验码

    # 码距

    • A码到B码需要改变的最小位数，码距越大，越利于纠错和检错

    # 分类

    • 奇偶校验码

      参考：https://zhuanlan.zhihu.com/p/26509678

      • 分为水平、垂直、水平垂直奇偶校验码
      • 编码中增加一位校验位，码距为2
      • 可检错但不能纠错
      • 奇校验一个重要的特征是产生不了全0代码

    • 循环冗余校验码（CRC）

      • 可检错但不能纠错，校验码越多，代码的校验能力越强

      • 生成多项式为k个数据，产生r个校验位编码，编码长度为k+r

      • 组成

        • 左边为数据，右边为校验码
        • 数据占k位，则校验码为n-k位，n为CRC码的字长

    • 海明校验码

      • 参考：海明校验码——如何求解 (opens new window)

      • 本质是奇偶校验码

        • 数据位是n位，校验位是k位

      • 编码规则

        • ① 确定数据位D和校验位Ｐ在海明码中的位置

          • Pi在海明码第2^(i-1)位，即海明码Hj=Pi，则j=2^(i-1)

        • ② 确定校验关系

          • 被校验的海明位的下标等于所有校验该位校验位的下标之和，而校验位由自身校验

        • ③ 检测错误

          • G4G2G1G0...奇校验全1，偶校验全0

    # 中央处理单元（CPU）

    • 组成

      • 运算器、控制器、寄存器组和内部总线

    • 功能

      • 实现程序控制、操作控制、时间控制、数据处理

    • 运算器

      • 组成

        • 算术逻辑单元ALU

          • 负责处理数据、算术运算、逻辑运算

        • 累加寄存器AC

          • 运算结果或源操作数存放区

        • 数据缓冲寄存器DR

          • 作为CPU和内存、外部设备之间的数据传送中转站，暂存内存的指令和数据

        • 状态条件寄存器PSW

          • 保存算术指令和逻辑指令运行或测试的结果建立的各种条件码内容，分为状态标志和控制标志

      • 功能

        • 执行所有的算术运算、逻辑运算和逻辑测试

    • 控制器

      • 指令控制逻辑

        • CPU依据指令周期的不同阶段区分二进制的指令和数据

        • 步骤

          • 取指、译指、执行指令、形成下一条指令地址

            首先将PC的指令地址取出，送入地址总线，CPU依据指令地址AR去内存中取出指令内容存入IR；而后由ID进行分析指令操作码，最后执行指令，取出指令执行所需的源操作数

        • 组成

          • 指令寄存器IR

            • 暂存CPU从内存储器取到缓存寄存器的指令

          • 程序计数器PC

            • 寄存信息和计数功能，分顺序执行和转移执行

          • 地址寄存器AR

            • 保存当前CPU所访问的内存单元地址

          • 指令译码器ID

            • 指令包含操作码和地址码。对指令操作码进行分析识别，向操作控制器发送控制信号

        • 指令寻址方式

          • 顺序寻址、跳跃寻址

        • 指令操作数的寻址方式

          • 立即寻址，直接给出操作数，而非地址。特点是指令执行时间很短，不需要访问内存取数。
          • 直接寻址，在指令格式的地址字段中直接指出操作数在内存的地址。
          • 间接寻址，给出的是指向操作数地址的地址 。
          • 寄存器寻址，当操作数不放在内存中，而是放在CPU的通用寄存器中时，可采用寄存器寻址方式。此时指令中给出的操作数地址不是内存的地址单元号，而是通用寄存器的编号。
          • 基址寻址，将CPU中基址寄存器的内容加上指令格式中的形式地址而形成操作数的有效地址。它的优点是可以扩大寻址能力。同形式地址相比，基址寄存器的位数可以设置得很长，从而可以在较大的存储空间中寻址 。
          • 变址寻址，给出的地址需与特定的地址值累加从而得出操作数地址。与基址寻址方式计算有效地址的方法很相似，它把CPU中某个变址寄存器的内容与偏移量相加来形成操作数有效地址。 但使用变址寻址方式的目的不在于扩大寻址空间，而在于实现程序块的规律性变化。
          • 相对寻址，把程序计数器PC的内容加上指令格式中的形式地址D而形成操作数的有效地址。采用相对寻址方式的好处是程序员无须用指令的绝对地址编程，所编程序可以放在内存任何地方。

      • 时序控制逻辑

        • 为每条指令按时间顺序提供应有的控制信号

      • 总线控制逻辑

        • 为多个功能部件服务的信息通路的控制电路

      • 中断控制逻辑

        • 用于控制各种中断请求，根据优先级高低进行排队

    • 寄存器组

      • 专用寄存器

        • 运算器和控制器中的寄存器是专用寄存器
        • 作用固定

      • 通用寄存器

        • 用途广泛，可由程序员规定用途
        • 数目因处理器不同有所差异

    • 指令的执行步骤

    # 指令系统

    # 分类

    • 精简指令系统RISC

      • 指令少、使用频率接近、主要依靠硬件实现

      • 寻址方式支持方式少

    # 指令的流水处理

    流水线原理

    • 取指、分析、执行

    指令控制方式

    • 顺序方式

      • 顺序串行执行，执行完一条在取下一条指令

    • 重叠方式

      • 在解释K条指令操作完成之前就可以解释第K+1条指令

    • 流水方式

      • 把重复的顺序处理过程分解若干过程，每个字过程能在专门独立模块有效并行工作

    RISC采用的流水技术

    • 超流水线

      • 在每个机器周期内能完成一个甚至两个浮点操作，以及时间换空间

    • 超标量

      • 内装多条流水线同时执行多个处理，以空间换时间

    • 超长指令字

      • 同时执行多条指令，发挥软件作用

    # 流水线时间计算

    • 流水线周期

      • 指令分成不同执行段，其中执行时间最长的段为流水线周期

    • 流水线执行时间

      • 1条指令总执行时间+（总指令条数-1）*流水线周期

    • 流水线总时间

      • （n条指令+2）*流水线周期

    • 单缓存区和双缓存区

      • 能够同时执行的阶段就是流水线的独立执行阶段；只能独立执行的阶段应该合并为流水线中的一个独立执行阶段

    • 流水线吞吐率计算

      • 单位执行的指令条数

        • TP=指令条数/流水线执行时间

      • 最大吞吐率

        • TPmax=1/#t

    • 流水线加速比计算

      • 即使用流水线后的效率提升，越高流水线效率越高
      • S=不使用流水线执行时间/使用流水线执行时间

    • 流水线效率

      • E=n个任务占用的时空区/k个流水段的总时空区（阴影面积）

    • 总结

      • 最大吞吐率取决于流水线中最慢一段所需时间；
      • 如果流水线出现断流，加速比会明显下降；
      • 要使加速比和效率最大化应该对流水线各级采用相同的运行时间；

    # 存储系统

    用于存放程序或数据的存储部件有CPU内部寄存器、高速缓冲存储器（Cache）、主存储器（内存储器、内存）、辅存（外存储器、外存）。它们的存取速度不一样，从快到慢依次为寄存器->Cache->内存->辅存。Cache是访问速度最快的层次。

    # 目的

    • 为了解决存储容量、成本和速度之间的矛盾问题

    # 分类

    • 按存储器位置

      • 内存、外存

    • 按存储器构成材料

      • 磁、半导体、光存储器

    • 按存储器工作方式

      • 可读可写、只读存储器

    • 按存储器访问方式

      • 按地址访问、按内容访问（相联存储器）

    • 按寻址方式

      • 随机存储器

        • 访问任意存储单元所用时间相同

      • 顺序存储器

        • 只能顺序访问，如磁带

      • 直接存储器

        • 二者结合，如磁盘，对于磁道的寻址是随机的，在一个磁道内侧是顺序的

    两级存储映像为

    • Cache主存、主存-辅存（虚拟存储体系）

    # 高速缓存Cache

    • 概念

      • 用来存储最活跃的程序和数据；直接和CPU交互；位于CPU和主存之间，容量小速度快。内容是主存内存的副本拷贝

    • 组成

      • 由控制部分和存储器组成

        • 控制部分判断CPU要访问数据是否在Cache中，在则命中，不则依据一定算法从主存中替换

    • 局部性原理

      • 定义

        • 总的来说，在CPU运行时，所访问的数据会趋向于一个较小的局部空间地址内（如循环操作，循环体被反复执行）

      • 分类

        • 时间局部性原理

          • 如果一个数据正在被访问，那么在近期它很可能会被再次访问，即在相邻的时间里会访问同一个数据项

        • 空间局部性原理

          • 在最近的将来会用到的数据的地址和现在正在访问的数据地址很可能是相近的，即相邻空间地址会被连续访问

    • 地址映射方法

      • 定义

        • 需要主存地址->Cache存储器地址，由硬件自动完成映射

      • 分类

        • 直接映像

          • 主存、Cache都分成块并编号
          • 主存中的块与Cache的块的对应关系是固定的
          • 地址变换不灵活，容易造成资源浪费
          • Cache的块号i和主存的块号j有函数关系：i=j%m（其中m为Cache总块数）

        • 全相联映像

          • 主存、Cache都分成块并编号
          • 主存中任意一块都与Cache中任意一块对应，可随意调入Cache任意位置
          • 地址变换复杂，速度较慢。因此，当Cache满了才会发生块冲突，是最不容易发生块冲突的映像方式
          • 主存地址/缓存地址=块号+块内地址

        • 组组相连映像

          • 主存、Cache先分块再分组
          • 两者结合，组间采用直接映像，即主存中组号与Cache组号相同的组才能命中；组内采用全相联映像，即组号相同的两个组内的所有块可以任意调换

      • 命中率和平均时间

        • 即当CPU所访问的数据在Cache中，命中，直接从Cache中读取数据，若不在，则需从内存中读取

        • 若读取一次Cache时间为1ns，从内存中读取时间为1000ns，在多次读取中，有90%命中Cache，则CPU读取一次平均时间为

        • （90%*1+10%1000）ns

    # 虚拟存储器

    • 定义

      • 将很大的数据分成许多较小的块，全部存储在外存中，无需考虑地址映像关系，系统自动完成

    • 基本层级

      • 主存—缓存—外存

    • 管理方式

      • 分为页式、段氏、段页式

    # 输入输出方式

    # 分类

    • 程序控制（查询）方式

      • CPU主动查询外设是否完成数据传输
      • 由程序主动查询外设，完成主机与外设间的数据传送，方法简单，硬件开销小。

    • 程序中断方式

      • 外设完成数据传输，向CPU发送中断，等待CPU处理，效率较高

      • 中断响应时间

        • 从发出中断请求到开始进入中断处理程序

      • 中断处理时间

        • 从中断处理开始到处理结束

      • 中断向量

        • 提供中断服务程序入口的地址

      • 多级中断嵌套

        • 使用堆栈来保护断点和现场

    • DMA方式（直接存储器存储方式）

      • CPU只需完成必要初始化操作，数据传输的整个过程都由DMA控制器（DMAC）完成，在主存和外设直接建立直接的数据链路，效率较高
      • DMAC和CPU共享系统总线，并且具有独立访问存储器的能力
      • 在一个总线周期结束后，CPU会响应DMA请求开始读取数据
      • CPU响应程序中断方式请求是在一条指令执行结束时
      • 区分指令执行结束和总线周期结束
      • DMA 传送方式的优先级高于程序中断，两者的区别主要表现在对CPU的干扰程度不同。中断请求不但使CPU停下来，而且要CPU执行中断服务程序为中断请求服务，这个请求包括了对断点和现场的处理以及CPU与外设的传送，所以CPU付出了很多的代价；DMA请求仅仅使CPU暂停一下，不需要对断点和现场的处理，并且是由DMA控制外设与主存之间的数据传送，无需CPU的干预，DMA只是借用了一点CPU的时间而已。

    • 输入/输出处理机

      • 也是一种处理机，内存具有独立处理系统，是数据传输独立于CPU

      • 通道是一种通过执行通道程序管理I/O操作的控制器，它使主机与I/O操作之间达到更高的并行程度。

      • 分类

        • 字节多路通道传送方式

          • 每一次传送一个通道的一个字节，多路通道循环

        • 选择通道传送方式

          • 选择一个通道，先传送完这个通道的所有字节，再开始下一个通道传送

    # 总线

    # 定义

    • 任何连接两个电子元器件的导线
    • 计算机系统的运算速度与CPU 中的数据总线宽度有关

    # 分类

    按功能分：总线的分类：数据总线（DB），地址总线（AB），控制总线（CB）

    按位置分：内部总线（处理器内部通信总线），外部总线（设备一级的总线，计算机外部设通信的总线）

    • 内部总线
    • 内部芯片级别，芯片与处理器之间通信的总线
    • 系统总线
      • 是板线总线，可分为数据总线（并行数据传输位数），地址总线（系统可管理的内存空间大小 ）、控制总线（传送控制命令。）
      • 代表的有ISA总线、EISA总线、PCI总线。
    • 外部总线
      • 与外设的总线
      • 代表的有Ｒ232（串行总线，适合长距离传输）、SCSI（并行总线，近距离高速数据传输）、USB（通用串行总线）
      • Centronic总线属于外部总线，它的接口是一种打印机并行接口标准，用于将计算机与打印机等外设相连接。
    • 将多个处理机组成多处理系统
      • STD总线即STD bus（Standard for an 8-bit microcomputer bus system），是一种规模最小、面向工业控制的8位系统总线，支持多处理器系统。
      • 交叉开关将各个CPU连接成动态互连网络，组成多处理机系统。
      • PCI（Peripheral Component Interconnect,外部组件互连）总线用于将显卡、声卡、网卡和硬盘控制器等高速外围设备直接挂在CPU总线上，其负责CPU和外围设备的通信。 过插在PCI插槽上的网卡可以组建局域网，所以，通过PCI总线是可以组建多处理机系统。

    # 总线计算

    • 总线的时钟周期=时钟频率的倒数
    • 总线的宽度（传输速率）=单位时间内传输的数据总量/单位时间大小

    # 系统可行性分析

    # 系统可靠性

    • 可以用平均故障时间MTBF来衡量

    • 系统任一时间能正常工作的概率公式

      • MTBF/(1+MTBF)，可用性

    • 平均无故障时间MTTF

      • MTTF/(1+MTTF)，可靠性

    • 平均故障修复时间MTTR

      • 1/(1+MTTR)，可维护性

    • 平均故障间隔时间MTBF

      • MTBF = MTTR + MTTF
      • 在实际应用中，一般MTTR很小，所以通常认为MTBF≈MTTF

    # 串联系统

    • R=R1*R2*…Rn

    # 并联系统

    • R=1-(1-R1)*(1-R2)*…(1-Rn)

    计算机系统性能的重要指标

    • 性能测评常用方法: 时钟频率 / 指令执行速度 / 等效指令速度 / 数据处理速率PDR；
    • 基准测试程序: 帧数测试程序 浮点测试程序 SPEC基准测试程序 TPC基准测试程序
    • 系统响应时间是指用户发出完整请求到系统完成任务给出响应的时间间隔。
    • 作业吞吐量是指单位时间内系统完成的任务量。
    • 若一个给定系统持续地收到用户提交的任务请求，则系统的响应时间将对作业吞吐量造成一定影响。若每个任务的响应时间越短，则系统的空闲资源越多，整个系统在单位时间内完成的任务量将越大；反之，若响应时间越长，则系统的空闲资源越少，整个系统在单位时间内完成的任务量将越小。

    # 加密技术

    • 对称加密

      • 对数据的加密和解密的密钥是相同的
      • DES、AES、3DES、RC-5、IDEA
      • 密钥一般56位，适合加密大数据

    • 非对称加密

      • 密钥不同，加密速度慢，计算量大
      • RSA、ECC、Elgamal
      • 密钥一般1024位
