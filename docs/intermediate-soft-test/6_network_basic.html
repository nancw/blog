    网络基础

    nanc 8/1/2021 软考

    # 网络基础

    # OSI模型

    # 七层

    # 应用层

    • 实现具体应用功能

      • HTTP、Telnet、FTP、SMTP（报文采用ASCII格式表示）、NFS

    # 表示层

    • 数据的格式与表达、加密、压缩

      • JPEG、ASCII、GIF、DES、MPEG

    # 会话层

    • 建立、管理和终止会话

      • RPC、SQL、NFS

    # 传输层

    • 端到端的连接

      • TCP、UDP、SPX

    # 网络层

    • 常见设备，路由器、三层交换机

    • 分组传输和路由选择

      • IP、RARP、ARP、ICMP、IGMP

    # 数据链路层

    • 常见设备，网卡、网桥、交换机

    • 传送以帧为单位的信息

      • IEEE 802.3/.2， HDLC， PPP， ATM等

    # 物理层

    • 常见设备，中继器、集线器

    • 二进制传输

      • RS232， V.35， RJ-45， FDDI等

    # 协议

    # 网络层

    • IP

      • 无连接不可靠

    • ARP、RARP

      • 地址解析协议，ip<-->mac

    • ICMP

      • 检测网络通信是否顺畅

    # 数据链路层

    • PPP

    # 应用层

    • DHCP

      • 动态分配IP地址

    • SNMP

      • 简单网络管理协议

    • DNS

      • 域名解析IP地址

        • 递归查询

          • 主机向本地域名服务器查询

        • 迭代查询

          • 本地域名服务器向根域名服务器查询

    # 网络技术标准和协议

    • 局域网协议

      • IEE802.3 标准以太网，10Mbps，传输介质同轴电缆
      • IEE802.3u 快速以太网，100Mbps，双绞线
      • IEE802.3z 千兆以太网，1000Mbps，光纤或双绞线
      • IEE802.4 令牌总线网、IEE802.5 令牌环网
      • 无线局域网CSMA/CA（载波侦听多路访问方法）

    • 广域网协议

      • 点对点协议PPP（拨号上网）

      • 数字用户线xDSL（ASDL上传网速和下载网速不对等，下载网速一般很快，传输介质电话线）

      • 数字专线DDN（市内或长途的数据电路）

        • 专线上网

      • 帧中继（以帧为传输单位）

    # TCP/IP协议族

    # 四层

    应用层（应用层、表示层、会话层）、 传输层、网际层（网络层）、 网络接口层（数据链路层、物理层）

    # ping

    # netstat

    # 设备

    # 交换机

    • 以太网交换机

      • 根据MAC地址进行交换

    • 帧中继（FR）交换机

      • 只能根据虚电路号DLCI进行交换

    • 异步传输（ATM）交换机

      • 根据虚电路标识进行信元交换

    • 三层交换机

      • 网络层设备

    • 二层交换机

      • 工作在数据链路层的设备 ， 不能划分网络层的广播， 即它的各个端口形成一个广播域

    • 一种多端口网桥

      • 数据链路层设备

    # 集线器（Hub）

    • 多端口中继器，所有端口组成一个冲突域
    • 连接网络线路的装置
    • 没有自动寻址的作用

    # 网桥

    • 连接两个局域网的存储转发设备，对接收信号进行再生放大

    # 路由器

    • 识别IP地址，对数据包进行转发

    # 路由选择策略

    # 静态路由选择

    • 不能找出最佳路由
    • 分固定式、洪泛式(扩散法)、随机路由选择3种

    # 动态路由选择

    • 利于改善网络性能，但算法复杂会增加网络负担
    • 分布式、集中式、混合式路由选择

    # windows server 2003的路由类型

    直连网络ID， 用于直接连接的网络，Interface（ 或nexthop） 可以为空 远程网络ID，用于不直接连接的网络，可以通过其他路由器到达这种网络Interface字段是本地路由器的IP地址 主机路由，到达特定主机的路由，子网掩码为255.255.255.255 默认路由， 无法找到确定路由时使用的路由，目标网络和网络掩码都是0.0.0.0 持久路由，利用route add –p命令添加的表项，每次初始化时， 这种路由都会加入Windows的注册表中， 同时加入路由表

    当Windows服务器收到一个IP数据据包时，先查找主机路由，再查找网络路由（ 直连网络和远程网络），这些路由查找失败时， 最后才查找默认路由

    # 计算机网络分类

    # 分布范围

    • 局域(LAN)、城域(MAN)、广域网(WAN)

    # 拓扑结构

    总线型，利用率低、干扰大、价格低 星型，交换机形成的局域网、中央单元负荷大 环型，流动方向固定、效率低扩充难 树型，总线型的扩充、分级结构 分布式，任意节点连接、管理成本高

    • 总线型、星型、环型、树型、分布式

    # 网络设计

    # 层次化局域网（层次化网络设计）

    • 分三层
      • 接入层、提供用户接入，由连接用户的低端交换机构成；
      • 汇聚层、网络访问策略、数据包处理、过滤、寻址等中级操作，由用于实现策略的路由器和交换机构成；
      • 核心层、负责高速数据交换，由经过高端路由机和交换机组成；
    • 设计思路，从下往上，向接入最后核心

    # 考虑因素

    • 可用性

      • 网络可用于执行预期任务时间所占总量百分比

    • 可靠性

      • 网络设备持续执行预定功能的可能性，用平均障碍间隔时间度量

    • 可恢复性

      • 网络从故障种恢复的难易程度和时间

    • 适应性

      • 用户改变应用要求事的网络应变能力

    • 可伸缩性

      • 网络技术或设备随用户需求增长而扩充的能力

    # IP地址

    特殊含义IP地址： 127网段，回播地址； 网络号全0，当前子网的主机； 全1地址，本地子网的广播； 主机号全1，特定子网的广播；

    10.0.0.0/8，10.0.0.1-10.255.255.254

    172.16.0.0/12， 172.16.0.1-172.31.255.254

    192.168.0.0/16， 192.168.0.1-192.168.255.254

    169.254.0.0， 保留地址，用于DHCP失效（Win） 0.0.0.0, 保留地址，用于DHCP失效（Linux）

    # 32位二进制数

    • 点分十进制表示法

    # 三级IP地址

    • 将IP地址分成，网络号、子网号、主机号

    # 两级IP地址

    • 主机号部分

      • 所占字长直接决定了所包含网络中最大的主机数

    • 网络号部分

      • 所占字长就直接决定整个互联网可以为多少个网络分配IP地址

    # 判断IP地址是否同一网段，确定网络号是否相同

    # 分类编址

    • IP地址的前4位来决定地址所属类别

      • A（0）、1.0.0.0--126.0.0.0，8位网络号，子网掩码255.0.0.0 B（10）、128.1.0.0--191.254.0.0，16位网络号，255.255.0.0 C（110）、192.0.1.0--223.225.254.0，24位网络号，255.255.255.0 D（1110）、224.0.0.0--239.255.255.255，组播 E（1111）类、254.0.0.0--245.255.255.255，保留
      • A类网络号占8位，主机号为32-8=24位，主机分配的主机数个数2^24-2，全0全1不能分配，全0代表网络、全1代表广播

    # VLSM/无分类编址

    • 一种产生不同大小的子网的网络分配机制
    • IP地址/网络及子网编码比特数表示。 如：192.168.123.0/26表示前26位为网络号和子网号， 即子网掩码为26位长， 主机号为6位长

    # 子网划分和子网掩码

    # 子网掩码

    • 网络号、子网号都为1，主机号为0
    • 利用子网掩码区分是二级还是三级IP地址

    # 子网划分

    参考：https://blog.csdn.net/dyyay521/article/details/94381876

    • 子网号可全0和全1，主机号不能全0和全1

    • 超网

      • 聚合网络为超网，划分子网的逆过程，将网络号取出几位做主机号
      • 超网的子网掩码如何计算

    # 网络号与主机号的计算

    • ip地址与子网掩码and运算得到网络地址
    • 子网掩码与IP地址and运算得到主机地址

    # IPv6

    # 特性

    • 地址长度，128位
    • 简化报文头部格式，加快报文转发，提高吞吐量
    • 提高安全性、身份认证和隐私权
    • 允许协议继续转变，增加新功能

    # 地址分类

    • 单播、类似IPv4的ABC类地址 任播、类似4的主机号全1广播地址 组播、类似4的D类地址

    # IPv4到6过渡采用的基本技术

    • 双协议栈、隧道技术、翻译技术

    # 其他概念

    # NAT、默认网关、VLAN、VPN

    # PPP(二次握手)

    # 冲突域和广播域

    • 路由器可阻断广播和冲突域，交换机只能阻断冲突域
    • 一个路由器可划分多个广播和冲突域
    • 一个交换机整体是一个广播域，但可划分多个冲突域
    • 物理层设备集线器下整体作为一个冲突和广播域

    # 网络安全

    # 各层安全保障

    • 物理层采用物理隔离，其他层采用协议保障传输安全

    # SSL

    • 三方面服务

      • 用户和服务器合法验证、加密数据、保护数据完整性

    • 实现过程

      • 接通阶段->会谈阶段->检验阶段->客户认证阶段->结束阶段

    # 防火墙

    • 网络防火墙

      • 包过滤、状态检测
      • 层次低，但效率高

    • 应用防火墙

      • 双穴主机、屏蔽主机、屏蔽子网
      • 解决伪装的危险包，层次高、效率低
      • 屏蔽子网只能屏蔽外网攻击，内网攻击无法屏蔽

    # 病毒

    • 病毒，一组计算机指令或程序代码
    • 传染性、隐蔽性、潜伏性、破环性、针对性、衍生性、寄生性、未知性
    • 系统引导性、文件外壳型、目录型、蠕虫病毒、宏病毒、CIH病毒(破坏硬件)

    # 木马

    • 一种后门程序
    • QQ消息尾巴、特洛伊木马
    • 红色代码，是蠕虫病毒+木马

    # 网络攻击

    • 主动攻击
    • 被动攻击

    # 综合布线系统

    # 工作区子系统

    • 由终端设备连接到信息插座的连线组成，包括连接器和适配器。调制解调器不属于此子系统

    # 水平干线子系统

    • 实现信息插座和管理子系统间的连接，星型结构

    # 管理间子系统

    • 由交连、互连配线架组成，主要设备配线架、HUB和机柜、电源

    # 垂直干线子系统

    • 负责连接管理间子系统到设备间子系统的子系统

    # 设备间子系统

    • 将公共系统设备的各种不同设备互连，由设备间电缆、连接器和相关支撑硬件组成

    # 楼宇（建筑群子系统）
