    # 数据结构

    # 线性结构

    # 线性表存储结构

    # 顺序存储结构

    • 读O(1)，插入O(n/2),删除O([(n-1)/2])

    # 链式存储结构

    • 读O([(n-1)/2])，插入删除为O(1)

    # 队列和栈

    队列先进先出，栈先进后出

    # 双端队列

    • 具有队列和栈的性质

    # 循环队列

    循环队列Q的定义rear表示队尾元素的指针， len表示队列的长度，fear表示队列的队头指针

    • # 队头元素指针计算公式

      • (rear-len+1+M)%M

    • # 队列中元素个数

      • (rear-fear+M)%M

    • # 队满的判断

      • front=(rear+1)%M

    # 散列存储结构

    将节点按其关键字的散列地址存储到散列表中。常用的散列函数有除余法、基数转换法、平方取中法、折叠法、移位法和随机数法等

    # 数组、矩阵、广义表

    # 二维数组存储位置计算

    • 设每个数据元素占用L个单元，m、n为数组的行数和列数

      • 行 Loc（aij）=Loc（a11）+（（i-1）×n+（j-1））×L

      • 列 Loc（aij）=Loc（a11）+（（j-l）×m+（i-1））×L

    • 注意数组下标的开始值

    # 广义表

    • 是线性表的推广，线性表的元素的是不可分的单元素；广义表的元素既可以是单元素，也可以是有结构的表；

    # 树

    # 树的概念

    • # 度

      • 为一个结点子树的数目，如A的度为3，B的度为2，C的度为0，D的度为1；
      • 总度数为结点树减1；

    • # 树的度

      • 树中各结点的度的最大值

    • # 树的高度

      • 一棵树的最大层数，如上图的树的高度为3；

    • # 树的层次、深度、高度

      • 树中各结点的层次的最大值

    • # 父结点、双亲结点

      • 上对于下端的结点

    • # 孩子结点

      • 下对于上端的结点

    • # 兄弟结点

      • 同一父结点的多个子结点

    • # 叶子结点--终端结点

      • 度为0的结点，如E,F,C,G为叶子结点

    • # 分支结点

      • 除叶子结点外的所有结点

    • # 内部结点

      • 根以外的分支结点，如B,D为内部结点

    # 单支树

    • 每个节点只有一个孩子

    # 树的遍历

    • # 前序遍历

      • 根-左-右

    • # 中序遍历

      • 左-根-右

    • # 后序遍历

      • 左-右-根

    • # 层次遍历

      • 层层访问

    # 树转二叉树规则

    • 树中某结点M的孩子结点， 在生成二叉树后放在M结点的左孩子位置；
    • 而M的兄弟结点， 在生成二叉树后放在M结点的右孩子位置；

    二叉树形态

    • # 满二叉树

      • 深度为k且有2^k-1（ k≥1） 个结点

    • # 完全二叉树

      • 深度为k、有n个结点的二叉树中各结点能与深度k的顺序编号的满二叉树从1到n标号的结点对应
      • 适合采用顺序存储结构

    • # 非完全二叉树

    • # 大顶堆、小顶堆

      • 满足堆的含义，即完全二叉树中所有非终端节点的值均不大于（或者不小于）其左、右孩子节点的值。其他序列不满足此定义，如（10,18,15,20,50,80,30,60）转换为二叉树如下图：

    # 二叉树性质

    有5个度为1的结点, 4个度为2的结点， 2个度为3的结点， 那么 树中总的度数为5+8+6=19， 因此树中除根节点外， 就应该有19个节点， 所以树中总的节点数应该为20， 那么叶子节点数=20-5-4-2=9

    • 第i层上最多有结点( i≥1）
    • 深度为k的二叉树最多有结点（ k≥1）
    • 对任何一棵二叉树， 如果其叶子结点数为n0， 度为2的结点数为n2， 则n0=n2+1

    完全二叉树

    • 具有n个结点的完全二叉树的深度为

      • 编号i的节点： 若i=1时， 该节点为根节点， 无双亲 若i>1时， 该节点的双亲节点为(i/2) 若2i≤n，该节点的左孩子编号为2i， 否则无左孩子 若2i+1≤n，该节点的右孩子编号为2i+1,否则无右孩子

    # 哈夫曼树（最优二叉树）

    https://blog.csdn.net/qq_29519041/article/details/81428934

    • 不是完全二叉树， 也不是平衡二叉树

    • # 编码长度

    • # 编码

      • 左-0 右-1

    • # 概念

      • 树的路径长度

        • 从树根到树中每一结点的路径长度之和

      • 权

        • 在一些应用中，赋予树中结点的一个有某种意义的实数

      • 结点的带权路径长度

        • 结点到树根之间的路径长度与该结点上权的乘积

      • 树的带权路径长度( 树的代价)

        • 所有叶结点的带权路径长度之和

      • 在结点数目相同的二叉树中，完全二叉树的路径长度最短；

    • 使用的是贪婪法，不可能存在度为1的分支结点

    • 总的结点数是2n-1的结点， 而叶子结点数正是n

    # 平衡二叉树

    • 左右两个子树的高度差的绝对值不超过1，如满二叉树、完全二叉树

    # 线索二叉树

    若前驱结点不为空， 而且其右指针域为空， 则将根结点的地址赋给前驱结点的右指针域， 并将前驱结点的右线索标志置1； 若根结点的左指针域为空， 则把前驱结点的地址赋给根结点的左指针域， 同时将根结点的左线索标志置1； 将根结点地址赋给保存前驱结点指针的变量， 以便当访问下一个结点时， 此根结点成为前驱结点

    • # 如二叉排序树或空树

      • 若左子树不空，且左子树上所有结点值均小于根结点值
      • 若右子树不空， 则右子树上所有结点值均大于根结点值
      • 左右子树也都是二叉排序树
      • 中序遍历， 必定得到结点关键字的有序序列
      • 空链域存放指向结点在遍历过程中的直接前驱或直接后继的指针
      • 为了使在二叉排序树上进行的查找操作性能最优， 构造二叉排序树时需要进行平衡化处理， 使每个节点左、 右子树的高度差绝对值不超过1

    # 图相关概念

    # 有向图、无向图、连通图

    # 无向完全图

    • 限定任何一条边的两个顶点都不相同， 则有n个顶点的无向图至多有n(n-1)/2条边

    # 有向完全图

    • 恰好有n(n-1)条边的有向图称为有向完全图

    # 度、入度和出度

    • 度：入度和出度之和

    • 入度：表示以该顶点为终点的有向边的数目

    • 出度：表示以该顶点为起点的有向边的数目

    • # 完全图适合采用邻接矩阵法

    • # 无向图顶点的度，为直接与该顶点相邻的顶点个数

    # 图的存储结构

    # 邻接矩阵表示法

    • n个顶点，nxn矩阵
    • A[i,j]表示i到j结点是否有边，存在为1，不存在为0
    • 无向图的邻接矩阵是对称的，可采用压缩存储法
    • 利用邻接矩阵容易求得各个顶点的度： 无向图Vi的度为第i行(列)的元素之和； 有向图Vi的出度第i行元素之和，入度是第i列元素之和

    # 邻接表表示法

    • n个顶点，n个链表，除头结点外，其他存放为邻接结点号
    • n个顶点e条边 无向图，需要n个头结点，2e个表结点 有向图，需要n个头结点，e个表结点
    • 无向图顶点Vi的度，是第i个单链表的结点数 有向图顶点Vi的出度，是第i个单链表的结点数，入度必须遍历所有链表

    # 深度优先 --DFS

    • 类似树前序遍历

    # 广度优先 --BFS

    • 类似树层次遍历

    # 最小生成树

    # 连通图G的子图是一棵包含G所有顶点的树，则该子图称为G的生成树

    # 代价最小的生成树

    # 求连通的带权无向图的最小代价树的常用算法

    • 普里姆算法（ Prim算法）
      • 从起点开始把当前拥有的节点最短的路径和节点载入。
    • 克鲁斯卡尔算法（ Kruskal算法）
      • 不管节点，直接按顺序载入最小的边，当连起来时就是最小生成树。

    # 拓扑排序

    • 依次遍历没有前驱结点的结点。 而某一时刻没有前驱结点的结点有可能存在多个， 所以一个图的拓扑排序可能有多个

    # AOV网

    • 表示节点的顺序关系，是有向无回路图.即如果a->b,那么a是b的先决条件，求这种拓扑序列就是AOV

    # AOE网

    • 带权的有向无环图 AOE网是以边表示活动的有向无环网，在AOE网中，具有最大路径长度的路径称为关键路径，关键路径表示完成工程的最短工期

    # 最短路径

    • 单源最短路径，迪杰斯特拉算法；
    • 多源最短路径，弗洛伊德算法；
