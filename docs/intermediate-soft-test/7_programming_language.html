    # 程序设计语言

    # 程序设计语言概述

    # 特点

    • 低级语言

      • 机器语言，识别0和1，如汇编语言

    • 高级语言

      • 抽象级别更高

    • 函数式程序设计语言，的控制结构取决于函数，以及函数的定义和调用。函数式语言主要用于符号数据处理，如微分和积分演算、数理逻辑、游戏推演及人工智能等其他领域；

    • 逻辑式程序设计语言，编写程序不需要描述具体的解题过程，只需要给出一些必要的事实和规则。这些规则是解决问题的方法的规范说明，根据这些事实和规则，计算机利用谓词逻辑，通过演绎得到求解问题的执行序列。这种语言主要用在人工智能领域，也应用在自然语言处理、数据库查询、算法描述等方面，尤其适合于作为专家系统的开发工具。

    • 过程式语言，也叫做命令式语言或强制式语言，是通过指明一列可执行的运算及运算的次序来描述计算过程的语言。例如：FORTRAN,ALGOL 60,COBOL,PASCAL,C等都是过程式语言。

    # 解释和编译

    • 将高级语言翻译成机器语言

    • 编译

      • 生成独立可执行文件，直接运行，效率高

    • 解释

      • 不生产可执行文件，逐条解释执行，用于调试模式，执行速度慢

    # 组成

    • 语法（一组规则）、语义（语法成分含义）、语用（构成语言的各个记号和使用者关系）
    • 语句用于描述程序中的运算步骤、控制结构及数据传输。
    • 语法是语言的外观，给出语言的语法意味着给出语句、声明和其他语言结构的书写规则。
    • 语义则表示不同的语法结构的含义。
    • 语用是关于程序与使用者之间的关系。
    • 语境是理解和实现程序设计语言的环境，包括编译环境和运行环境。

    # 程序设计语言基本成分

    # 数据成分

    • 分类

      • 常量、变量、全局量、局部量

    • 数据类型

      • 整型、字符型、双精度、单精度浮点型、布尔型等

      • 数据必须具有类型

        • 有利于在翻译程序的过程中为数据合理分配存储单元
        • 有利于对参与表达式计算的数据对象进行检查
        • 有利于规定数据对象的取值范围及能够进行的运算

    # 运算成分

    • 允许使用的运算符即运算规则

      • 算数、逻辑、关系、位运算等

    • 逆波兰式

      • 也叫后缀表达式，即将运算符写在操作数之后的表达式，它不需使用括号，在将算术表达式转换为逆波兰式表示时，需要分配2个栈，一个作为临时存储运算符的栈S1（含一个结束符号），一个作为输入逆波兰式的栈S2（空栈）
      • 逆波兰式ab-cd+转换为中缀表达式的过程为：ab-cd+ = （ab-）*（cd+） = （a-b） （cd+） = （a-b）（c+d）

    • 表达式

      • 前缀表达式对应于二叉树的前序遍历
      • 中缀表达式对应于二叉树的中序遍历
      • 后缀表达式对应于二叉树的后序遍历

    # 控制成分

    • 结构

      • 顺序结构
      • 选择结构
      • 循环结构
        • while先判断再执行，所以循环体的执行次数要比判断次数少1；
        • do-while先执行再判断，进行判断的次数和执行循环体的次数刚好相等；

    • 语句

      • 复合语句
      • if和switch语句
      • 循环语句

    # 传输成分

    • 指明语言允许的数据传输方式。如：赋值处理、数据的输入输出等

    # 函数

    • 函数使用涉及三个概念：函数定义、函数声明（先声明后使用）、函数调用

    # 调用

    • 传值调用：将实参的值传递给形参，形参的改变不会导致调用点所传的实参的值改变。实参可是是合法的变量、常量、表达式
    • 传址调用：即引用调用，将实参的地址传值给形参，即相当于实参存储单元的地址引用，因此其值改变的同时就改变了实参的值，实参不能为常-量，只能是合法的变量和表达式

    # 汇编

    # 三类语句

    • 指令语句

      • 将其汇编后能产生相应机器代码。这些代码能被CPU识别并执行相应操作
      • 产生机器代码，操作必须在程序运行时完成
      • 必须具有操作码字段，可以没有操作数字段

    • 伪指令语句

      • 指示汇编程序在汇编源程序时完成某些工作，如变量分配存储单元地址，给某符号赋值
      • 不产生机器代码，在源程序汇编时完成

    • 宏指令语句

      • 允许用户将多次重复使用的程序段定义为宏，宏的引用

    # 概念

    • 词法分析

      • 对源程序字符进行扫描根据构词规则识别单词（也称单词符号或符号）

    • 语法分析

      • 逻辑阶段，根据识别的单词组合成各类语法短语，如：程序，语句，表达式等，分析判断源程序在结构是否正确

      • 分析方法

        • 自上而下分析

          • 最左推导，从左到右

        • 递归下降思想

          • 是一种自上而下的方法

        • 自下而上分析

          • 最右推导，从右至左

        • 移到-规约思想

          • 自下而上的方法

    • 语义分析

      • 逻辑阶段，对结构上正确的源程序进行上下文有关性质的审查。如：类型匹配、除法除数不为0等。分为静态语义错误（编译阶段能够发现）、动态语义错误（运行时能发现）

    • 中间代码和目标代码

      • 中间代码是语义分析产生的需要经过优化链接，最终生成可执行的目标代码。
      • 引入中间代码的目的是进行与机器无关的代码优化处理。
      • 常用的中间代码有后缀式（逆波兰式）、三元式（三地址码）、四元式和树等形式

    • 中间代码优化和目标代码生成

      • 代码生成需考虑的问题
        • 中间代码形式、目标代码形式、寄存器分配、计算次序选择

    • 源程序中可能出现的错误两类：语法错误和语义错误。

      • 词法错误如非法字符或拼写错关键字、标识符等； 语法错误是指语法结构出错，如少分号、begin/end不配对等；
      • 静态语义错误：如类型不一致、参数不匹配等； 动态语义错误（逻辑错误）：如无穷递归、变量为零时作除数等 ；

    # 语言及文法

    • 语言

      • 语言是按照一定规则排列的符号和集合。要形式化地描述一个语言，就需要借助文法的概念。

    • 文法

      • 文法就是用来描述语言的语法结构的形式规则

      • 四元组

        • G=（V,T,S,P） V: 不是语言组成部分，可理解为占位符；非终结符：能够推导出其他元素 T: 终结符，语言组成部分，是最终结果；不能推导出其他元素 S: 七十符，文法开始符号 P：产生式，用终结符替代非终结符的规则；即非终结符推导出终结符的公式

      • 闭包

        • 一般考察闭包可以为0个的代入运算

      • 类型

        • 程序设计语言中的大部分都是上下文无关文法，当然语义是相关的

    • 文法和语言的形式描述

      • 字母表、字符串、字符串集合及运算

    • 总结

      • 用正规式来定义一些简单的语言，但是很多复杂一些的语言不能用正规式表达。例如，正规式不能用于描述配对或嵌套的结构，具体的例子有：由配对括号构成的串的集合不能用正规式描述，语句的嵌套结构也不能用正规式描述。还有，重复串也不能用正规式表示，例如，集合{wcw|w是a和b的串}不能用正规式描述。
      • 正规式只能表示给定结构的固定次数的重复或者没有指定次数的重复。
      • 有限状态自动机识别的语言是正规语言。对于每个非确定的有限自动机，都有一个与其等价的正规式。
      • 上下文无关文法的描述功能比正规式更强，正规式可以描述的每种结构都可以用上下语言无关文法来描述，但反之不然。上下文无关文法能表示次数不固定的重复。
