    # 操作系统

    # 概述

    # 作用

    • 通过资源管理提高计算机系统的效率
    • 向用户提供良好的人机界面

    # 特征

    • 并发性、共享性、虚拟性、不确定性

    # 功能

    • 进程、存储、文件、设备、作业管理

    # 分类

    • 批处理操作系统、分时操作系统（轮流使用CPU工作片）、实时操作系统（快速响应）、网络操作系统、分布式操作系统（物理分散的计算机互联系统）、微机操作系统（windows）、嵌入式操作系统

    # 嵌入式系统初始化过程

    • 自底向上

    • 硬件到软件的次序为

      • 片级初始化->板级初始化->系统初始化

        芯片级是微处理的初始化，板卡级时其他硬件设备初始化，系统级初始化时软件及操作系统初始化

    # 微内核操作系统

    • 系统分为用户态和内核态

    • 单核

      • 方便进程切换，但内核庞大，稳定性弱

    • 微内核

      • 稳定性高，便于裁剪，但进程间切换消耗资源，效率低

    # 文件管理

    # 索引文件结构

    • 分类

      块长为512B， 每个块号占3B， 一个物理块可放： 512/3=170个目录项， 也即： 一个一级索引可存放的文件大小为： 170×512=87040B。 一个二级索引可存放文件的大小为： 170×170×512=148×105B。 一个三级索引可存放文件的大小为： 170×170×170×512=251×107B。

      • 直接索引

        • 存放的是内容

      • 一级间接索引

        • 存放的是链接到物理盘块的地址

      • 二级索引

        • 存放的是一级地址

    # 文件和树形目录结构

    • 文件安全等级

      • 系统安全（不允许未经授权的用户）、用户安全（指定用户分配访问权）、目录安全（与用户权限无关）、文件安全（与用户权限无关）

    • 树形结构

      • 区分相对路径和绝对路径

    • 全文件名

      • 等于绝对路径+文件名

    • 文件物理结构

      • 连续结构、链接结构，索引结构。

    • 文件目录是文件控制块FCB（数据结构）的有序集合。

    • 文件存取方式

      • 顺序存取，随机存取、按键存取。

    # 文件存储空闲空间管理

    外存空间管理的数据结构为磁盘分配表法。

    • 空闲区表法

      • 空闲空间整合成一张表，即空闲文件目录

    • 空闲链表法

      • 空闲空间链接成一个链表，根据需要分配

    • 成组链接法

      • 既分组，每个组内又链接成链表，上述方法综合

    • 位示图法

      若系统的字长为32位， 磁盘上的物理块依次编号为： 0、 1、 2、 …， 那么4096号物理块的使用情况在位示图中的第____个字中描述。若磁盘的容量为200GB， 物理块的大小为1MB， 那么位示图的大小为____个字 答： ①（ 4096+1） /32等于128余1，为129 ② 200GB/1MB=200 ×1024个 物理块， 需要200 ×1024位来表示， 因此位示图中字的个=200×1024/32=6400个

    • 对每个物理空间用一位标识，为1则使用，为0则空闲

    # 进程管理

    # 组成

    • 进程控制块PCB

      • 唯一标志

    • 程序

      • 描述进程要做什么

    • 数据

      • 存放进程执行时所需数据

    # 进程状态图

    • 作用

      • 用于展现进程的状态，以及各种状态之间的转换

    • 常见模型

      • 三态模型

        • 系统自动控制时只有三种状态

      • 五态模型

        • 人为操作才会进入状态，活跃就绪即就绪，活跃阻塞即等待

      • 七态模型

    # 前趋图和进程资源图

    • 前趋图

      • 用来表示哪些任务可以并行执行，哪些任务之间有顺序关系

    • 进程资源图

      • 用来表示进程和资源之间的分配和请求关系

      • P代表进程、R代表资源，R指向P表示R有资源已分配给P，P指向R表示P还需请求一个R资源才能执行

      • 阻塞节点

        • 某资源所请求的资源已全部分配完毕，无法获取所需资源，该进程阻塞无法继续

      • 非阻塞节点

        • 某进程所请求资源还有剩余，可分配给该进程继续执行

      • 死锁

        • 但所有进程都是阻塞节点时，即陷入死锁状态

      • 资源图化简方法

        • 把不阻塞的进程所有边都去掉，形成孤立的点，再把分配给这个进程的资源回收，再看剩下进程哪些不阻塞，然后逐个变成孤立点，最后，所有资源和进程都变成孤立的点

    # 进程间通信

    • 同步和互斥

      • 同步

        • 表示两任务可同时执行，但速度上有差异，需速度上匹配，不存在资源是否单独或共享的问题

      • 互斥

        • 表示一个资源在同一时间内只能由一个任务单独使用需要加锁，使用完后解锁才能被其他任务使用

      • 临界区

        • 指进程对临界资源实施操作的那段程序，本质时一段程序代码

        • 对互斥临界管理原则

          • 有空即进、无空则等、有限等待、让权等待

    • 信号量机制

      • 分类

        • 同步信号量

          • 对共享资源的访问控制，初值不定

        • 互斥信号量

          • 对临界资源采用互斥访问，使用后其他进程无法访问，初始值为1

      • PV操作

        • 原子操作，用来解释进程间的同步和互斥原理

        • V操作

          • 将某资源+1

            • 若相加结果大于零，进程继续执行
            • 若相加结果小于或等于零， 则唤醒一阻塞在该信号量上的进程， 然后再返回原进程继续执 行或转进程调度

        • P操作

          • 先将某资源-1

            • 若相减结果大于等于零， 则进程继续执行
            • 若相减结果小于零， 则阻塞一个在该信号量上的进程， 然后再返回原进程继续执行或转进 程调度

    # 进程调度

    • 定义

      • 指当有更高优先级的进程到来时如何分配CPU

    • 三级调度

      • 高级调度

        • 又称长调度或作业调度，决定哪个作业可调入系统中

      • 中级调度

        • 又称对换调度，决定哪个就绪进程可以调入内存中

      • 低级调度

        • 又称进程调度，决定内存中哪个就绪进程可以占用CPU

    • 调度算法

      • 先来先服务FCFS

        • 先到达进程优先分配CPU。用于宏观调度

      • 时间片轮转

        • 分配每个进程CPU时间片，轮流使用CPU，每个进程时间大小相同。用户魏观调度

      • 优先级调度

        • 每个进程都有一个优先级，优先级大的先分配CPU

      • 多级反馈调度

        • 时间片轮转和优先级调度相结合

    • 死锁问题

      • 定义

        • 两个以上的进程都要求对方已经占有的资源导致无法继续运行下去的现象

      • 产生的四个必要条件

        • 资源互斥，每个进程占有的资源并等待其他资源、系统不能剥夺进程资源、进程资源图是一个环路

      • 打破四大条件

        • 死锁预防

          • 限制并发进程对资源的请求

        • 死锁避免

          • 采用银行家算法避免，提前计算出一条不会死锁的资源分配方法，分配资源，否则不分配资源

        • 死锁检测

          • 允许死锁产生，当系统定时运行一个检测死锁程序，若检测到，则设法加以解除

        • 死锁资源计算

          • 系统内n个进程，每个进程需要R资源， 那么发生死锁的最大资源数为n*(R-1)， 不发生死锁的最小资源数为n*(R-1)+1

    # 进程和线程

    • 进程属性

      • 可拥有资源的独立单位，最小单位
      • 可独立调度和分配的基本单位

    • 线程属性

      • 是独立调度的最小单位
      • 可共享进程的公共数据、全局变量、代码、文件等资源，但不能共享线程独有资源，如线程栈指针等标识数据

    # 存储管理

    # 存储器结构

    • 寄存器--高速缓存Cache--主存--外存

    # 地址重定位

    • 定义

      • 将逻辑地址转换为主存物理地址的过程

    • 分类

      • 静态重定位

        • 在程序装入主存时完成转换

      • 动态重定位

        • 边运行边转换

    # 分区存储管理

    • 定义

      • 将某进程运行所需的内存整体一起分配，再执行

    • 分类

      • 固定分区（静态）

        • 主存分若干固定分区，将运行作业装配进去
        • 分区固定，大小和作业需要大小不同，会产生内部碎片

      • 可变分区（动态）

        • 定义

          • 主存空间分区时在作业转入时划分，正好划分为作业需要的大小，不存在内部碎片，
          • 容易将主存空间切割成许多块，产生外部碎片

        • 算法

          • 首次适应法

            • 按内存地址顺序从头查找

          • 最佳适应法

            • 将内存中空闲内存块从小到大排序

          • 最差适应法

            • 将内存中空闲块最大的
            • 预防系统中产生过多细小空闲块

          • 循环首次适应法

            • 按内存地址顺序查找，不每次从头查找

          • 可重定位分区

            • 解决碎片问题，移动已经分配好区域，使其成为连续区域

    • 缺点

      • 但进程运行所需内存大于系统内存时，就无法将整个进程一起调入内存

    # 页式存储管理

    • 解决空间极大的进程运行问题

    • 优点

      • 利用率高，碎片小，分配及管理简单

    • 缺点

      • 增加系统开销，可能产生抖动现象

    • 逻辑页

      • 页号|页内地址（物理偏移地址）

    • 页表

      • 页号与物理块号并非按序对应，需查询页表，逻辑页号对应物理块号

    # 快表

    • 定义

      • 本质也是页表，只是页表的存储位置在Cache中

    • 可快速知道逻辑页号对应的物理地址，加快转化速度

    # 段式存储管理

    • 定义

      • 将进程空间分配成一个段，每个段有段号和段内地址
      • 每段物理大小不同，分段式根据逻辑整体结构分段

    • 段表

      • 有段表和基址两个属性，才能确定一个逻辑段在物理段的位置

    • 与页式区别

      • 分页式根据物理空间划分，每页大小相同；分段式根据逻辑结构划分，每段是完整功能，便于共享，但大小不同

    # 段页式存储管理

    • 对进程空间先分段，后分页

    • 优点

      • 空间浪费小，存储共享容易，保护容易，能动态连接

    • 缺点

      • 复杂性和开销增加，需要硬件及占用内容增加，故执行速度大大下降

    # 虚拟存储管理

    # 页面置换算法

    • 缺页

      • 表示要执行的页不在内存物理中，需从外部调入内存，增加执行时间。
      • 缺页越多，系统执行效率越低
      • 在淘汰页面时， 应选择页帧号最大的进行淘汰

    • 算法

      • 最优算法OPT

        • 理论上的，原理是选择未来最长时间内不被访问的页面置换

      • 先进先出算法FIFO

        • 先调入内存的页先置换淘汰
        • 产生抖动现象，分配的页数越多，缺页率越多

      • 最近最少使用LRU

        • 过去最少使用的页面被置换淘汰
        • 效率高，不产生抖动现象

      • 淘汰原则

        • 优先淘汰最近未访问的，而后淘汰最近被修改的页面

    # 设备管理

    # 分类方式

    • 按数据组织分类

      • 块设备、字符设备

    • 资源分配角度

      • 独占设备、共享设备、虚拟设备

    • 数据传输速率

      • 低速、中速、高速设备

    # I/O控制

    • I/O设备管理软件所有层次及每一层功能

    • 采用DMA方式，可使设备与主存间的数据块传送不需要CPU干预

    # DMA

    • 实现DMA传送的基本操作

      实现DMA传送的基本操作如下： （ 1） 外设可通过DMA控制器向CPU发出DMA请求： （ 2） CPU响应DMA请求， 系统转变为DMA工作方式， 并把总线控制权交给DMA控制器； （ 3） 由DMA控制器发送存储器地址， 并决定传送数据块的长度； （ 4） 执行DMA传送； （ 5） DMA操作结束， 并把总线控制权交还CPU

    # 虚设备和SPOOLING技术

    • 采用SPOOLING技术

      • 在外设上建立两个数据缓冲区，分为输入井和输出井

    • 利用磁盘设备提供虚拟设备

    # 磁盘调度

    • 磁盘结构和参数

      • 每个盘面有多个同心圆，每个同心圆是一个磁道，每个同心圆又划分多个扇区，数据放在扇区

        • 柱面号->扇区号->磁头号

      • 寻道时间

        • 指磁头移到到磁道所需时间

      • 等待时间

        • 等待读写扇区转到磁头下方所用时间

      • 存取时间

        • 寻道时间+等待时间

    • 调度算法

      • 先来先服务FCFS

        • 根据进程请求访问磁盘的先后顺序

      • 最短寻道时间优先SSTF

        • 远处进程可能永远无法访问
        • 根据请求访问的磁道与当前磁道最近的进程优先调度

      • 扫描算法SCAN

        • 磁头在磁盘双向移到，选择离磁头当前所在磁道最近的请求访问磁道，且与磁头移到方向一致（里到外，外到里）

      • 单向扫描调度算法CSCAN

        • 磁头是单向移动的， 也就是当磁头从内向外移动到最外面时， 磁头放到最内， 然后再从内向外扫描

    • 磁盘冗余阵列技术RAID

      • RAID0

        • 没有提供冗余和错误修复技术

      • RAID1

        • 在成对的独立磁盘上产生互为备份的数据，提供读取性能

      • RAID2

        • 将数据条块化的分布于不同硬盘上，并使用海明码校验

      • RAID3

        • 使用奇偶校验，并用单块磁盘存储奇偶校验信息

      • RAID5

        • 在所有磁盘上交叉的存储数据及奇偶校验信息

    # 作业管理

    # 作业四种状态

    • 提交、后备、执行（作业调入系统中执行，与进程执行状态类似）、完成

    # 作业调度算法

    • 优先级

      • 按优先级决定执行哪个作业

    • 先来先服务FCFS

      • 先来先执行

    • 短作业有限SJF

      • 按作业时间长度决定，运行短优先执行

    • 响应比高者优先HRN

      • 响应公式

        • (等待时间+运行时间)/运行时间

      • 响应高者优先执行

    • 定时轮转

      • 时间片轮转，定义以CPU时间片长度，作业执行完这个长度就停止等待，直到轮到

    # 作业周转时间

    • 单个作业的周转时间

      • 作业进入系统时间点-作业执行完的时间点
      • 或作业的等待时间+作业执行时间

    • 作业平均周转时间

      • 所有作业的周转时间/作业总个数

    • 单个作业的带权周转时间

      • 作业的周转时间/作业的实际运行时间

    • 作业平均带权周转时间

      • 所有作业的带权周转时间和/作业总个数
