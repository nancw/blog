    博客改进空间

    目前通过配置达到基本需求，后续有时间再做如下完善：

    • 文档名称命名不能有中文和空格；

    • 首页美化，只显示一幅图片，打字机等；

    • 侧边栏展开不太流畅；

    • 侧边栏无需配置自动显示信息；

    • 文档侧边栏优化，如针对手机端显示的侧边栏显示文章二级标题；

    • 侧边栏点击可隐藏；

    • 导航分类只需要配置front matter可分类后续无需在配置文件进行配置，但导航文档需在config.js配置才能分类，后续看是否有优化空间，即无需配置则可分类；

    • 导航栏文档按钮要配置readme的文件才能进入，否则404；

    • 网站整体样式修改；

    • 添加其他插件，如看板娘；

    • 代码主题的更改，当前为默认主题；markdown中prism无法支持代码块标记为c++形式的代码高亮，需标记为cpp；

    • 加密功能；

    • 配置文件格式整理；

    • 能否做成阅读模式、pdf文件显示；

    • markdown文档中尖括号编译报错，必须包含用``修饰；

    • markdown排版美化，目前发现加粗字体的显示不够友好；

    • 文章某些参考未做详尽整理，有时间再一一标明；

    • 目录跳转电脑端无法跳转，但手机端正常跳转；

    • 页内跳转跳转不了；

    • 页内跳转只能从下往上跳转，不能从上往下跳（typora编辑器的问题）;

    • 文档目录栏网络编程刚开始点击无法进入，需点击两次(图片后缀webp解析出粗导致)；

    • 图片名称不能有空格，否则会找不到，后续可能考虑采用图床，重新整理图片命名规范；
